# CachyOS Smoke Test For 2.5.3

Use this on a fresh CachyOS VM before publishing the release.

## 1. Prepare the VM

Install the runtime and packaging dependencies:

```bash
sudo pacman -Syu
sudo pacman -S --needed base-devel git go npm mpv gtk3 webkit2gtk-4.1 unzip chromium
```

## 2. Copy the repo into the VM

Work from a Linux-native path, not directly from a shared folder:

```bash
mkdir -p ~/work
cp -r ~/vmshare/miruro/. ~/work/miruro
cd ~/work/miruro
```

## 3. Verify the app builds on CachyOS

```bash
rm -rf build frontend/node_modules
npm --prefix frontend install
go run github.com/wailsapp/wails/v2/cmd/wails@v2.11.0 build -clean -platform linux/amd64 -tags webkit2_41
```

Expected result:
- build completes
- `build/bin/Nipah! Anime` exists

## 4. Run the Arch packaging gateway

```bash
bash scripts/verify-arch-pkgbuild-local.sh
```

Expected result:
- `makepkg` completes successfully
- local Arch package is produced from the current repo state

## 5. Install and launch the package

If the gateway succeeded, do one direct packaging/install check too:

```bash
cd ~/work/miruro/packaging/arch
rm -rf src pkg
makepkg -si
which nipah-anime
nipah-anime
```

Expected result:
- package installs cleanly
- `nipah-anime` launches

## 6. Smoke-test the release behavior

Inside the app:
- Home loads
- Anime Online opens
- AnimePahe landing pages and playback work
- AnimeGG landing pages load
- AnimeGG playback works on at least one subbed title and one dubbed title
- Local Anime still shows completed downloads/imported entries

## If the Arch gateway fails

Do not publish yet.

Check:
- `packaging/arch/PKGBUILD`
- `build/bin/PKGBUILD`
- LF line endings in Linux packaging files
- version alignment in `wails.json`, `frontend/package.json`, `packaging/arch/PKGBUILD`, and `packaging/linux/nfpm.yaml`

Rebuild the release artifacts after fixing the repo, then rerun this smoke test from step 3.
