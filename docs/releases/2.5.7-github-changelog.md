# Nipah! Anime 2.5.7

## Fixed
- Fixed the public source package used by `PKGBUILD` so Linux builds no longer miss the performance debug helper source file.
- Fixed the public source release path for Arch-based users so `PKGBUILD` builds from a complete GitHub tag tarball.
- Fixed custom `MPV` path handling so both a direct executable path and a real install folder work correctly.
- Fixed the `Settings` page black-screen regression caused by the removed theme controls.
- Fixed Linux online playback failures caused by invalid MPRIS/DBus track metadata during external player startup.

## Improved
- Direct episode download remains available through `AnimePahe` and `AnimeGG`.
- Downloaded episodes continue to create entries in `Local` automatically.
- `AnimePahe` remains the primary stable English fallback source.
- `AnimeGG` keeps the faster, higher-quality path where the upstream host is healthy.

## Hidden For Stability
- `AnimeKai` remains hidden from the public source picker until its playback path is stable enough to ship.
- The unfinished public light/dark theme toggle remains hidden from the release UI.

## Known Notes
- `AnimeGG` still depends heavily on upstream host health. Some episodes or audio variants may fail or behave inconsistently even when the app path is correct.
- On Linux, `AnimePahe` currently expects a Chromium-based browser for its browser-backed anti-bot flow.

## Release Validation
- `go build .`
- `go test ./backend/...`
- `npm run build`
- Windows installer rebuilt from the current repo state.
- Linux `.deb` and AppImage rebuilt from the current repo state.
- `PKGBUILD` updated for `2.5.7` and intended for a fresh public CachyOS smoke test against the new tag.
